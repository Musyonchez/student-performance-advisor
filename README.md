# Student Performance Advisor - Knowledge-Based System

A rule-based expert system that evaluates a student's academic data and recommends whether they are **passing**, **at risk**, or **require intervention**.

---

## Problem Being Solved

Lecturers and academic advisors often need to identify struggling students early. This KBS automates that process: given a student's grade, attendance, assignment submission rate, daily study hours, and prior failure history, the system reasons through a set of rules and produces a conclusion with a clear explanation.

---

## How to Run

```bash
# Install Python 3.10+, then:
python main.py
```

To generate the semantic network diagram (requires matplotlib):

```bash
pip install matplotlib

```

### Phase 1 - Classify Inputs
| Rule | Condition | Result |
|---|---|---|
| R01 | grade ≥ 75 | academic_performance = good |
| R02 | 50 ≤ grade < 75 | academic_performance = average |
| R03 | grade < 50 | academic_performance = poor |
| R04 | attendance ≥ 80% | attendance_status = good |
| R05 | 60% ≤ attendance < 80% | attendance_status = average |
| R06 | attendance < 60% | attendance_status = poor |
| R07 | assignments ≥ 90% | assignment_status = good |
| R08 | 70% ≤ assignments < 90% | assignment_status = average |
| R09 | assignments < 70% | assignment_status = poor |
| R10 | study_hours ≥ 4 | study_status = sufficient |
| R11 | 2 ≤ study_hours < 4 | study_status = moderate |
| R12 | study_hours < 2 | study_status = insufficient |

### Phase 2 - Determine Conclusion
| Rule | Condition | Conclusion |
|---|---|---|
| R13 | good grade + good/average attendance | PASS |
| R14 | poor grade + poor attendance | FAIL |
| R15 | good grade + poor attendance | AT RISK |
| R16 | average grade | AT RISK |
| R17 | poor grade + good/average attendance | AT RISK |

| Rule | Condition | Recommendation |
|---|---|---|
| R18 | study_status = insufficient | Increase study hours |
| R19 | attendance_status = poor | Improve attendance |
| R20 | assignment_status = poor | Complete assignments |
| R21 | prior failures + not passing | Seek counselling |
| R22 | passing + grade ≥ 85 | Consider advanced modules |

## How Inference Works

The system uses **forward chaining** in three phases:

1. **Classify** - raw inputs (grade, attendance, etc.) are mapped to status labels (good/average/poor) using threshold rules.
2. **Conclude** - status labels are combined to produce one of the three possible conclusions.
3. **Recommend** - additional rules fire to generate targeted advice based on weak areas.

Every rule that fires is logged and printed as a reasoning trace so the user can see exactly how the conclusion was reached.

---

## Possible Conclusions

| Conclusion | Meaning |
|---|---|
| PASS - Performing Well | Student is on track; no immediate concerns |
| AT RISK - Needs Improvement | Student is borderline; action needed |
| FAIL - Requires Intervention | Student is in a critical situation; urgent support required |

---

## Knowledge Representation

**Frame Representation** is defined in `knowledge_base.py` (`STUDENT_FRAME`). Each student is modelled as a frame with slots for raw inputs and derived facts, with `if_needed` facets that trigger the inference engine.

**Semantic Network** is generated by running `generate_diagram.py`, saved to `diagrams/semantic_network.png`.
